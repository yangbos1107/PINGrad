# 欢迎来到 PIN Grad

PIN Grad 是一个围绕 PIN Program（Portland Institute, NJUPT）成员构建的 ECE 方向 MS 申请与发展知识站。

它由 PIN Program 第一届毕业生发起，受到 [CS Grad](https://csgrad.com/) 与 [Open CS](https://opencs.app/) 的启发，但定位更加聚焦：  

> 专注 ECE 专业  
> 面向 PIN 项目学生  
> 强调可执行、可复用、可追踪  

我们不追求“大而全”的百科式收录，而是希望建立一个长期演进的、结构化的申请参考体系。

---

## 我们为什么要做这个

在准备研究生申请的过程中，我们发现：

- 经验分散在聊天记录、表格、个人笔记与零散群聊中；
- “成功案例”往往只展示结果，而不拆解路径；
- 很多决策（选校、定位、录取难度判断）缺乏统一口径；
- 后来者往往需要重复踩同样的坑。

作为 PIN Program 第一届学生，我们希望为之后的学生：

- 建立一个基础的参考框架；
- 提供统一结构的数据口径；
- 降低信息不对称带来的不必要焦虑；
- 让申请从“凭感觉”走向“有结构的决策”。

---

## 项目定位

PIN Grad 专注于：

- 🎯 **ECE 专业 MS 申请**
- 🌍 覆盖全球五大区域（北美 / 欧洲 / 亚洲 / 澳洲 / 其他）
- 🏫 涵盖 60+ 所高校项目（持续更新）
- 📊 提供 DataPoints（DP）与结构化申请背景字段
- 📈 提供按地区组织的录取难度导航（US/UK Zone，其他地区顺序展示）

这不是一个学校排名网站。

我们不会评价“学校好坏”。

我们关注的是：

> 在特定背景条件下，某类申请者进入某项目的难度与定位区间。

---

## 关于“录取难度导航（Zone / 区域排序）”的说明

PIN Grad 当前采用分区域导航策略：

- 美国、英国：使用 `Zone V` 到 `Zone I` 表示录取难度区间（`Zone V` 最难，`Zone I` 相对容易）
- 香港、新加坡、澳洲：取消分级，按地区内 ECE 专业排序顺序展示
- 不做跨地区难度映射（不同地区申请池和偏好不同，不能直接横向对比）

请注意：

- 区间 **不代表学校质量**
- 区间 **不等于综合排名**
- 区间 **不是官方定义**

它只是基于：

- 录取难度
- 背景匹配度
- 项目定位
- 历史数据趋势

所形成的一个“申请难度区间参考”。

其中：

- 美国分区的基础排序参考 USNEWS ECE 相关专排
- 英国分区的基础排序参考 QS 相关学科/项目排名
- 后续会结合持续积累的 DP 进行动态校准

你可以把它理解为：

> 申请决策时的风险分布图，而不是价值评判体系。

## 与其他站点的差异

PIN Grad 受到 CS Grad 与 OpenCS 的启发，但我们强调：

- 更聚焦：只讨论 ECE 方向 MS
- 更场景化：围绕 PIN Program 的真实背景结构
- 更结构化：统一数据字段，减少描述歧义
- 更过程化：记录申请决策逻辑，而非只看 offer 结果
- 更长期：入学不是终点，后续发展路径同样重要

---

## 你可以从哪里开始

1. 浏览 [DataPoints 总览](/datapoints)，了解我们如何记录背景与结果。
2. 进入 [申请跟踪](/tracker)，使用流程模板构建自己的策略。
3. 按 [地区导航](/docs/us) 查找与你背景最接近的区间参考。
4. 阅读对应区间或学校下的案例，理解其决策逻辑与路径。

---

## 如何贡献

PIN Grad 是一个学生自发维护的长期项目。

欢迎通过以下方式参与：

- **PR**：修正文案、补充案例、更新模板。
- **Issue**：报告过期信息、提出结构优化建议。
- **数据提交**：通过表单提交申请结果与背景字段（维护者会统一格式后入库）。
