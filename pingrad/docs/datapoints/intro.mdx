# DataPoints 总览

DataPoints 是 PIN Grad 的核心数据层，用于统一记录申请相关事实。

## 字段分层

- Profile：背景信息（学校、方向、课程、实习、科研）。
- Target：目标信息（地区、项目类型、预算、时间窗口）。
- Process：过程信息（投递批次、面试、等待时间）。
- Outcome：结果信息（admit/reject/waitlist 与后续去向）。

## 口径约定

- 时间一律使用 `YYYY-MM`。
- 项目名称优先使用官方英文名。
- Offer 需区分“正式录取”和“有条件录取”。

## 推荐使用方式

1. 先记录真实过程，再做分析结论。
2. 每次更新只改动新增事实，避免回填猜测。
3. 在复盘中只引用可验证字段。
