# 申请数据

申请数据覆盖选校、投递、面试、结果回收四个阶段。

## 最小可用字段

- Program Name
- Deadline
- Submission Date
- Interview Status
- Final Decision

## 追踪节奏建议

1. 每周更新一次状态。
2. 截止日前 14 天执行完整检查。
3. 每个结果记录“原因假设 + 后续行动”。

## 复盘框架

### 输入侧

检查背景、选校范围、材料质量是否匹配。

### 过程侧

检查投递节奏、沟通响应、面试准备是否稳定。

### 输出侧

不只看录取数量，也看结果质量和后续可选空间。
